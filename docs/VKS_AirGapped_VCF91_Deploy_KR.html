SECTION 01

개요

인터넷에 연결되지 않은 격리 환경에서 VKS 클러스터를 배포하기 위한 전체 접근 방식을 설명합니다.

3

에어갭 환경의 VKS 배포란

외부 인터넷 접근 없이, 필요한 아티팩트를 내부로 옮겨 배포하는 방식
  • 문제 — VKS는 OVA·컨테이너 이미지·패키지 등 다수 아티팩트를 온라인 저장소에서 받아온다. 보안·규제상 인터넷이 차단된 환경에서는 이 경로가 막힌다.
  • 접근 방식 — 인터넷에 연결된 Bastion 호스트에서 모든 아티팩트를 내려받아, 격리망 내부의 Admin 호스트로 물리적으로 복사한 뒤, VCF 내장 Software Depot에 업로드하여 Supervisor / VKS가 이를 소비하도록 한다.
  • 9.1.0의 핵심 개선 — VCF 9.1.0에는 OCI 레지스트리가 내장되어, 이전 릴리스에서 필요하던 별도의 외부 OCI 레지스트리 구축이 불필요해졌다.
    • 이전 릴리스는 외부 OCI 레지스트리 + 레거시 가이드 경로를 따라야 한다
    • 본 가이드는 VCF / VVF 9.1.0 전용

핵심 용어 (1/2) — 호스트

배포에 참여하는 두 개의 리눅스 VM 역할
Internet-facing

Bastion 호스트

  • 인터넷 연결 Linux VM
  • OVA·CLI·이미지·패키지 등 모든 아티팩트 다운로드 담당
  • 다운로드한 파일을 Admin 호스트로 반출(복사)
Air-Gapped

Admin 호스트

  • 에어갭 내부의 격리된 Linux VM
  • 배포 컨트롤 센터 — kubectl / VCF CLI 실행 지점
  • 권장 사양 : 2 vCPU · 4GB RAM · 150~200GB 여유 스토리지

Step 1 — 다운로드 @ Bastion

인터넷 연결 Bastion에서 네 종류의 아티팩트를 수집한다
  • ① K8s Release OVA — wp-content.broadcom.com, v1.34.2 기본. 최신 3개 버전 확보 권장
  • ② VCF CLI 9.1.0 — Broadcom Support Portal
  • ③ Supervisor Services — 이미지 + YAML (예: ArgoCD v1.1.0)
  • ④ VKS Standard Packages — tar (3.6.0-20260211)
Supervisor Services 다운로드
# 이미지 + YAML 을 로컬로 내려받기
$ python3 oci_image_depot_migrator.py download \
  --service argocd --version v1.1.0
Standard Packages tar
$ wget https://.../standard-packages-3.6.0-20260211.tar
VMware by Broadcom
13

Step 2–4 — Supervisor · Content Library · Namespace @ vCenter

vCenter 측 사전 구성 세 단계
  • Step 2 · Supervisor 활성화 — 네트워킹·스토리지 정책을 먼저 구성한 뒤 vCenter에서 Supervisor를 활성화한다.
  • Step 3 · 로컬 Content Library 생성 — 로컬(비게시) Content Library를 만들고 다운로드한 K8s Release OVA를 업로드한다. 이 라이브러리가 클러스터 노드 이미지 소스가 된다.
  • Step 4 · vSphere Namespace 생성 — 워크로드 클러스터가 위치할 네임스페이스를 생성한다 (예: ns01). 스토리지 정책·VM Class·권한을 이 네임스페이스에 연결한다.
i
이 세 단계는 모두 vCenter UI / Supervisor 측 작업으로, 이후 Admin 호스트에서 진행할 CLI 작업의 전제 조건이 된다.
VMware by Broadcom

Step 5 — Admin 호스트 구성 @ Admin

격리망 내부 배포 컨트롤 센터 셋업
  • kubectl 설치 — Supervisor에서 vsphere-plugin.zip 획득
  • vCenter 루트 CA 등록 — TLS 신뢰 체인 구성
  • 로그인 — vcf context create / use
  • Depot OCI 업로드 활성화 — 이관을 위해 일시적으로 enable
컨텍스트 생성 & 전환
$ vcf context create --name airgap \
  --endpoint supervisor.vcf.local
$ vcf context use airgap
Depot OCI 업로드 활성화
$ ./toggle_software_depot_oci_image_upload.sh enable
VMware by Broadcom
15

Step 6 — Depot 업로드 @ Depot

아티팩트를 내장 OCI 레지스트리로 이관
  • 업로드 대상 — Supervisor Services + VKS Standard Packages
  • 도구 — oci_image_depot_migrator.py upload
  • 완료 후 필수 — OCI 업로드를 즉시 disable
업로드 실행
$ python3 oci_image_depot_migrator.py upload \
  --input ./argocd-v1.1.0
$ python3 oci_image_depot_migrator.py upload \
  --input ./standard-packages-3.6.0
!
보안 필수 — Depot OCI 업로드는 무인증(unauthenticated) 상태로 열린다. 이관이 끝나면 반드시 disable 하라.
./toggle_software_depot_oci_image_upload.sh disable
VMware by Broadcom

Step 7 — Harbor 구성 @ Admin

VCF Automation이 없을 때 이미지 프록시로 Depot 경유
  • 조건 — VCF Automation 미사용 시 관리 프록시를 구성한다
  • 프록시 등록 — manage-depot-image-proxy.sh add
  • YAML 수정 — Harbor 이미지 참조를 프록시 경유로 변경 후 설치
이미지 프록시 등록
$ ./manage-depot-image-proxy.sh add
Harbor YAML 이미지 경로 (프록시 경유)
# before
image: registry.example.com/harbor/...
# after — depot-image-proxy 경유
image: depot-image-proxy.kube-system
  .svc.cluster.local/harbor/...
VMware by Broadcom
17

Step 8 — VKS 클러스터 배포 @ Admin

워크로드 클러스터 생성 + Standard Packages 설치
  • VKS 업데이트 — VKS 3.6.3으로 업데이트
  • 클러스터 생성 — vksConfig.yaml 적용
  • 애드온 설치 — vcf addon install (cert-manager 등)
  • 실패 시 — privileged 라벨 부여 + ReplicaSet 삭제
클러스터 생성 & 애드온
$ kubectl create -f vksConfig.yaml -n ns01
$ vcf addon install create cert-manager
애드온 설치 실패 시 조치
$ kubectl label ns <ns> \
  pod-security.kubernetes.io/enforce=privileged
$ kubectl delete rs <replicaset> -n <ns>

운영 포인트 5가지

현장에서 가장 자주 문제가 되는 지점
  • 1 · 9.1.0 최대 개선 — 내장 OCI 레지스트리로 외부 레지스트리 구축 불필요. 아키텍처가 단순해졌다.
  • 2 · 보안 필수 — Depot OCI 업로드는 무인증이므로, 이관 직후 즉시 disable 한다.
  • 3 · 파일 이동 순서 — 인터넷 → Bastion → Admin → Depot → Supervisor. 한 방향 흐름을 지킨다.
  • 4 · 프록시 경로 — 프록시 사용 시 YAML 이미지 경로를 depot-image-proxy 기준으로 수정한다.
  • 5 · 검증 BOM 준수 — vCenter 9.1.0 / VKS 3.6.3 / K8s 1.34.2 / Packages 3.6.0-20260211 / CLI 9.1.0 조합을 지킨다.
