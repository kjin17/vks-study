핵심 이점 & 활용 사례

고객이 얻는 가치와 대표 도입 시나리오
Benefits

핵심 이점

  • 익숙한 운영 — vCenter·DRS·vSAN·NSX 그대로
  • 개발자 셀프서비스 — 네임스페이스→온디맨드
  • 엔터프라이즈 보안 — vPod 격리·CIS/STIG·OIDC
  • 검증된 안정성 — CNCF + Broadcom 검증 애드온
  • 경제성 — 기존 HW·라이선스 재사용, 비용 showback
Use Cases

이런 고객에게

  • 앱 현대화 — 레거시 VM + 신규 컨테이너 통합
  • AI/ML — Custom Zone + 지능형 배치로 GPU 격리
  • Telco/5G — SR-IOV·DPDK 보조 NIC로 CNF 고성능
  • 멀티테넌시/CSP — 테넌트별 키·다중 네트워크·500 스케일
  • 규제·에어갭 — 인터넷 차단 환경 안전 배포

구성요소 맵핑 — K8s ↔ vSphere

Kubernetes 개념vSphere 구성요소매핑 설명
Control PlaneSupervisor VM (in vCenter)vCenter 내 프로비저닝, K8s 컨트롤 플레인 역할

Supervisor가 배포하는 3가지 워크로드

전통 VM부터 하이퍼바이저 네이티브 Pod, 완전한 게스트 클러스터까지
VM Services

전통 VM을 K8s API로

  • 기존 VM을 K8s API(YAML)로 관리
  • VM Operator CRD (VirtualMachine)
  • VM 생명주기 자동화
vPod

vSphere Pods

  • ESXi 위에서 직접 실행되는 네이티브 Pod
  • VM 격리 수준 보안·별도 Worker VM 불필요
  • 빠른 프로비저닝·높은 성능
VKS Cluster

Guest / TKG 클러스터

  • 독립적 업스트림 K8s 클러스터
  • CP+Worker 모두 VM·표준 K8s API 호환
  • 멀티 테넌트·Helm/kubectl

VCF 9.1 신규 기능 & 성능·스케일
